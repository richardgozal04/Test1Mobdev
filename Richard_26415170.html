Rangkuman perintah perintah git

  1. Git init : untuk menginisialisasi repository kosong di /.git/
  2. Git status : untuk melihat status dari repository jika tidak ada file yang masuk di tahap staging maka akan muncul list file di untracked files , jika tidak ada file sama sekali di repository maka akan muncul tulisan “Nothing to commit”
  3. Git add “nama file” : menambah file ke tahap staging
  4. Git commit –m : untuk memasukkan file ke dalam repository dari tahap staging
  5. Git log : untuk melihat history dari langkah langkah yang kita lakukan
  6. Git remote add “nama remote” “alamat repositori” : untuk menhubungkan local dengan remote server yang dimiliki git
  7. Git push –u “nama remote” master : untuk memasukan data local ke remote
  8. Git pull : untuk mengsinkronisasi antara local dan remote
  9. Git diff : untuk melihat perbedaan commit terbaru dan yang sebelumnya
  10. Git reset : untuk mengunstage file yang sedang di dalam tahap staging
  11. Git checkout : untuk mengembalikan file ke masa sebelum di commit , bisa juga dipakai untuk pindah ke branch lain
  12. Git branch clean_up : untuk mengclear branch
  13. Git rm : untuk menghapus file

    a. VCS Lokal

    Merupakan sebuah basis data local untuk menyimpan semua perubahan pada berkas yang berada dalam cakupan local

    b.  VCS Terpusat

    Merupakan sebuah basis data untuk menyimpan semua perubahan pada berkas yang dapat berbagai orang mengerjakan sehingga banyak orang dapat berkolaborasi bersama sama dalam mengerjakan

    c. VCS Terdistribusi

    Merupakan sebuah basis data untuk menyimpan semua perubahan pada setiap repository.jadi, jika server mati, dan sistem berkolaborasi melalui server tersebut, maka klien manapun dapat mengirimkan salinan repositori tersebut kembali ke server. Setiap checkout pada DVCS merupakan sebuah backup dari keseluruhan data
